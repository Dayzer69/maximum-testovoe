Предположения:

В связи с тем, что сайт https://www.example.com является условным и не содержит реальной функциональности авторизации, при разработке автотестов были сделаны следующие предположения:

- Страница входа доступна по адресу: `https://www.example.com/login`.
- На главной странице присутствует кнопка "Login", ведущая на страницу авторизации.
- Страница авторизации содержит следующие элементы:
  - Поле ввода имени пользователя с локатором `id="user_name"`;
  - Поле ввода пароля с локатором `id="password"`;
  - Кнопку отправки формы с локатором `id="submit"`;
  - Область отображения ошибки с локатором `id="error_box"`.
- При вводе некорректных данных отображается сообщение об ошибке: "Invalid credentials".
- При успешной авторизации пользователь перенаправляется на главную страницу сайта.
- После успешного входа отображается приветственное сообщение в формате: "Welcome, {username}".
- После авторизации становится доступна кнопка "Logout", позволяющая выйти из системы.
- После выхода из системы пользователь перенаправляется на главную страницу, и кнопка "Login" снова становится доступной.
- В тестах используются условные корректные учетные данные: user / password.
- В тестах используются условные некорректные учетные данные: test / test.
- Предполагается отсутствие дополнительных механизмов безопасности.


Ограничения:

- Сайт `https://www.example.com` является фиктивным и используется исключительно в демонстрационных целях. Реальная функциональность авторизации на нём отсутствует.
- Локаторы элементов и поведение системы основаны на предположениях и могут отличаться в реальном приложении.
- Тесты реализованы для браузера Google Chrome. Для запуска в других браузерах требуется дополнительная настройка WebDriver.
- Тесты зависят от пользовательского интерфейса (UI), поэтому изменения в верстке или атрибутах элементов могут привести к их падению.
- В тестах используются статические тестовые данные, отсутствует интеграция с реальной базой пользователей.
- Не рассматриваются сценарии, связанные с безопасностью, например, восстановление пароля.
- Тесты требуют стабильного интернет-соединения для корректного выполнения.
- Запуск тестов предполагается вручную.
- Отсутствует параллельный запуск тестов и кроссбраузерное тестирование.
- Отчёт о тестировании формируется с использованием Allure и требует установленного Allure CLI.
